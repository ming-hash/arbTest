## 三、服务器部署（Linux）

### 3.1 环境准备

```bash
# 1. 安装 Python 3.11
sudo apt update
sudo apt install python3.11 python3.11-venv

# 2. 安装 Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# 3. 创建项目目录
mkdir -p /opt/arbdashboard
cd /opt/arbdashboard
```

### 3.2 部署后端

```bash
# 1. 创建虚拟环境
python3.11 -m venv .venv
source .venv/bin/activate

# 2. 安装依赖
cd backend
pip install -r requirements.txt

# 3. 使用 systemd 管理
sudo tee /etc/systemd/system/arbdashboard.service << EOF
[Unit]
Description=ArbDashboard Backend
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/opt/arbdashboard/backend
Environment="PATH=/opt/arbdashboard/.venv/bin"
ExecStart=/opt/arbdashboard/.venv/bin/python main.py

[Install]
WantedBy=multi-user.target
EOF

# 4. 启动服务
sudo systemctl daemon-reload
sudo systemctl enable arbdashboard
sudo systemctl start arbdashboard

# 5. 查看状态
sudo systemctl status arbdashboard
```

### 3.3 部署前端

```bash
# 1. 安装依赖
cd /opt/arbdashboard/frontend
npm install

# 2. 构建生产版本
npm run build

# 3. 使用 Nginx 提供静态文件
sudo tee /etc/nginx/sites-available/arbdashboard << EOF
server {
    listen 80;
    server_name your-domain.com;

    location / {
        root /opt/arbdashboard/frontend/dist;
        try_files \$uri \$uri/ /index.html;
    }

    location /api {
        proxy_pass http://localhost:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade \$http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host \$host;
        proxy_cache_bypass \$http_upgrade;
    }
}
EOF

# 4. 启用配置
sudo ln -s /etc/nginx/sites-available/arbdashboard /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## 四、数据库备份

### 4.1 手动备份

```bash
# 备份数据库
cp D:\Study\arbTest\database\arb_master.db D:\Study\arbTest\database\arb_master_backup_$(date +%Y%m%d).db
```

### 4.2 自动备份脚本

```python
# backend/utils/backup_db.py
import shutil
from datetime import datetime
from pathlib import Path

DB_PATH = Path(r"D:\Study\arbTest\database\arb_master.db")
BACKUP_DIR = Path(r"D:\Study\arbTest\database\backups")

def backup_database():
    """备份数据库"""
    BACKUP_DIR.mkdir(parents=True, exist_ok=True)
    timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
    backup_path = BACKUP_DIR / f"arb_master_backup_{timestamp}.db"
    shutil.copy2(DB_PATH, backup_path)
    print(f"数据库已备份到: {backup_path}")

if __name__ == "__main__":
    backup_database()
```

### 4.3 定时备份（Windows 任务计划程序）

```powershell
# 创建备份脚本 D:\Study\arbTest\backup.ps1
$timestamp = Get-Date -Format "yyyyMMdd_HHmmss"
$source = "D:\Study\arbTest\database\arb_master.db"
$dest = "D:\Study\arbTest\database\backups\arb_master_backup_$timestamp.db"
New-Item -ItemType Directory -Force -Path "D:\Study\arbTest\database\backups"
Copy-Item -Path $source -Destination $dest
```

### 6.3 API 性能监控

**使用 Python 装饰器**：
```python
import time
from functools import wraps

def monitor_performance(func):
    @wraps(func)
    async def wrapper(*args, **kwargs):
        start = time.time()
        result = await func(*args, **kwargs)
        elapsed = time.time() - start
        logging.info(f"{func.__name__} 耗时: {elapsed:.3f}秒")
        return result
    return wrapper
```

## 七、故障排查

### 7.1 常见故障清单

| 故障 | 可能原因 | 解决方案 |
|------|----------|----------|
| 后端无法启动 | 端口被占用 | 杀死占用 8000 端口的进程 |
| 前端无法加载 | 后端未启动 | 启动后端服务 |
| 数据不显示 | 数据库路径错误 | 检查 main.py 第41行路径 |
| 采样服务不运行 | 非交易时段 | 等待交易时段或手动触发 |
| 引擎无法启动 | 通达信未登录 | 登录通达信 |

## 八、安全建议

### 8.1 网络安全

1. **防火墙配置**：仅开放必要端口（80/443）
2. **HTTPS**：生产环境启用 HTTPS
3. **API 认证**：添加 JWT 或 API Key 认证

### 8.2 数据安全

1. **定期备份**：每天自动备份数据库
2. **加密存储**：敏感配置（如 QMT 路径）加密存储
3. **权限控制**：限制数据库文件访问权限

### 8.3 代码安全

1. **不提交敏感信息**：`.gitignore` 已配置
2. **输入验证**：使用 Pydantic 验证所有输入
3. **SQL 注入防护**：使用参数化查询

### 10.2 常见问题 FAQ

**Q1：为什么深度分析页面曲线不显示？**
A：等待采样服务运行至少 1 分钟，确认在交易时段。

**Q2：如何添加新的基金？**
A：在 `unified_fund_list` 表中 INSERT 新记录，或在"系统配置"页面管理。

**Q3：自动交易引擎如何暂停？**
A：在"信号监控"页面点击"停止引擎"按钮。

**Q4：数据库文件太大怎么办？**
A：清理 30 天前的 `fund_intraday_quotes` 数据。
