### 2.1 FundService（数据融合服务）

**文件路径**：`backend/services/fund_service.py`

**职责**：主页看板数据融合，从多个数据源获取并整合基金数据

#### 核心方法

##### `get_unified_dashboard_data()`

**功能**：获取主页看板的全量数据

**数据流**：
```
1. 查询 unified_fund_list 获取所有活跃基金
2. 对每只基金：
   a. 从 unified_fund_history 获取历史数据（最近10条）
   b. 从 fund_intraday_quotes 获取最新采样数据
   c. 从 fund_data 获取最新快照
   d. 调用 MarketDataService.get_realtime_quote() 获取实时价格
3. 融合数据，计算溢价率
4. 按分类（黄金原油/QDII欧美/QDII亚洲/国内LOF/白银）分组
5. 返回 JSON 数据
```

**重要说明：QDII欧美分类的特殊性**

前端 TAB "QDII欧美" 合并了三个内部类别：

**纯ETF（equity_us_etf）**：
- 代表基金：161126、161127、164906
- 跟踪标的：SPY、QQQ、GLD 等美股 ETF
- 估值算法：`calculate_etf_valuation` (ETF标的估值)

**美股指数（equity_us_index）⭐ 特殊处理**：
- 代表基金：161125、161130
- **跟踪标的**：.INX（标普 500）、.NDX（纳斯达克 100）
- **数据源**：**新浪指数价格**（非美股 ETF 价格）
- **时差处理**：中美交易时差约 13-15 小时，使用 T-1 日净值 + 当日实时指数价格
- 估值算法：`calculate_index_valuation` (指数标的估值，使用新浪指数数据)

**混合跨境（hybrid_cross）**：
- 代表基金：160225、501225、164824
- 跟踪标的：一篮子 ETF/股票
- 估值算法：`calculate_basket_valuation` (一篮子资产估值)

这种设计确保了：
1. **UI 简洁性**: 用户只需关注一个 "QDII欧美" TAB
2. **算法精确性**: 每种基金使用最适合的估值方法
3. **时差处理**: 美股指数类（161125、161130）的指数价格从新浪获取，考虑时差因素

**QDII欧美分类说明**：
- 前端 TAB "QDII欧美" 合并了三个内部类别：纯ETF、美股指数、混合跨境
- 虽然显示在同一个 TAB，但每种基金使用不同的估值算法
- 美股指数类（161125、161130）从新浪获取指数价格，考虑时差因素

**关键逻辑**：
- 静态溢价率 = `price / static_val - 1`
- 实时溢价率 = `price / rt_val - 1`
- 涨跌幅 = `(price / prev_close - 1) * 100`（现价除以前一日收盘价）
- 精度规范：A股价格3位、净值估值4位、美股标的2位

**数据来源说明**：
- **现价（price）**：[V4.0] 从实时接口（通达信/QMT）获取，毫秒级更新，用于套利计算；`fund_intraday_quotes`（60秒采样）仅用于画曲线图
- **成交量（volume）**：从实时接口获取（如有），或从 `unified_fund_history`（历史表）获取，单位：万元；若原始数据 > 100万，认为是股数，自动转换为万元
- **涨跌幅（price_change）**：`现价 / 前一日收盘价 - 1 * 100`，前一日收盘价从 `unified_fund_history` 表中获取

##### `get_fund_history(fund_code)`

**功能**：获取单基金的历史对账数据（用于深度分析页面）

**数据流**：
```
1. 查询 unified_fund_history 获取最近120条记录
2. 查询 fund_basket_weights 获取持仓权重
3. 查询 exchange_rate 获取历史汇率
4. 查询 usa_etf_daily_prices 获取历史标的价格
5. 计算每条记录的 pct_change（涨跌幅）
6. 计算估值误差（calculated vs actual）
7. 返回包含原材料列的数据
```

### 2.3 SamplerService（分时采样服务）

**文件路径**：`backend/services/intraday/sampler_service.py`

**职责**：后台守护线程，每分钟计算一次所有活跃基金的实时估值

#### 核心方法

##### `_perform_sample()`

**功能**：执行一次采样计算

**触发频率**：每 60 秒（仅 A 股交易时段）

**数据流**：
```
1. 判断当前时间是否在交易时段（9:30-11:30, 13:00-15:00）
2. 如果不是交易时段，跳过
3. 查询 unified_fund_list 获取所有活跃基金
4. 对每只基金：
   a. 调用 MarketDataService.get_realtime_quote() 获取实时价格
   b. 从数据库获取最新净值（nav）和净值日期（nav_date）
   c. 调用 DynamicValuationCalculator.calculate() 计算实时估值
   d. 计算溢价率 = (price - rt_val) / rt_val * 100
5. 批量 INSERT INTO fund_intraday_quotes
```

**防御逻辑**：
```python
# 过滤非股票代码（如 FINX、USDCNY）
import re
if not re.match(r'^\d{6}$', fund_code):
    continue  # 跳过非标准代码
```

**日志输出**：
```
[Sampler] 2026-06-08 10:30:00 - 采样 50 只基金
[Sampler] 162411: price=1.234, rt_val=1.200, premium=2.83%
[Sampler] 采样完成，写入 50 条记录
```

## 三、自动交易引擎

### 3.1 AutoTradeRunner（引擎运行器）

**文件路径**：`backend/core/auto_trade/engine_runner.py`

**职责**：守护线程，每 5 秒扫描全场，检查规则是否触发

#### 核心逻辑

```python
class AutoTradeRunner:
    def __init__(self):
        self.running = False
        self.rules = []
        self.positions = {}
        self.last_trigger = {}  # 冷却记录
    
    def start(self):
        """启动引擎"""
        self.running = True
        threading.Thread(target=self._run_loop, daemon=True).start()
    
    def _run_loop(self):
        """主循环"""
        while self.running:
            try:
                # 1. 获取全场实时溢价率
                dashboard_data = fund_service.get_unified_dashboard_data()
                
                # 2. 获取真实持仓（仅启动时同步一次）
                if not self.positions:
                    self.positions = trading_service.get_positions()
                
                # 3. 遍历规则
                for rule in self.rules:
                    if not rule['is_active']:
                        continue
                    
                    fund_code = rule['fund_code']
                    premium = self._get_premium(fund_code, dashboard_data)
                    
                    # 4. 检查是否触发
                    if self._should_trigger(rule, premium):
                        # 5. 执行下单
                        if rule['action'] == 'BUY':
                            trading_service.execute_order('BUY', fund_code, rule['order_vol'], price)
                        elif rule['action'] == 'SELL':
                            trading_service.execute_order('SELL', fund_code, rule['order_vol'], price)
                        
                        # 6. 记录日志
                        self._log_trade(rule, premium)
                        
                        # 7. 更新冷却记录
                        self.last_trigger[rule['id']] = time.time()
                
                time.sleep(5)  # 每 5 秒扫描一次
            
            except Exception as e:
                logger.error(f"引擎循环异常: {e}")
                time.sleep(10)  # 异常后等待 10 秒
```

### 3.2 RuleEngine（规则引擎）

**文件路径**：`backend/core/auto_trade/rule_engine.py`

**职责**：规则解析和匹配

#### 规则数据结构

```json
{
  "id": 1,
  "fund_code": "162411",
  "fund_name": "华宝油气",
  "direction": "BUY",           // BUY/SELL
  "trigger_type": "PREMIUM",    // PREMIUM（溢价触发）
  "threshold": -1.5,            // 折价 > 1.5%
  "order_vol": 10000,           // 单笔委托量（份）
  "max_pos_wan": 10,            // 仓位上限（万元）
  "capital_limit_wan": 5,       // 资金限额（万元）
  "is_active": true,
  "created_at": "2026-06-08"
}
```

#### 触发逻辑

```python
def should_trigger(rule, current_premium, current_position):
    """判断是否应该触发"""
    
    # 冷却检查
    last_time = last_trigger.get(rule['id'])
    if last_time and time.time() - last_time < 300:  # 300 秒冷却
        return False
    
    # 仓位检查
    if rule['direction'] == 'BUY':
        if current_position >= rule['max_pos_wan']:
            return False
    
    # 阈值检查
    if rule['trigger_type'] == 'PREMIUM':
        if rule['direction'] == 'BUY':
            return current_premium <= rule['threshold']  # 折价触发
        elif rule['direction'] == 'SELL':
            return current_premium >= rule['threshold']  # 溢价触发
    
    return False
```

## 五、后端启动流程

### 5.1 FastAPI Lifespan（启动钩子）

```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    # 启动时执行
    logger.info("系统启动中...")
    
    # 1. 初始化数据库
    db_manager.init_db()
    
    # 2. 启动分时采样服务
    sampler_service.start()
    
    # 3. 启动自动交易引擎
    auto_trade_runner.start()
    
    # 4. 启动 LOF011 数据同步（后台线程）
    threading.Thread(target=run_lof011, daemon=True).start()
    
    yield  # FastAPI 运行期间
    
    # 关闭时执行
    sampler_service.stop()
    auto_trade_runner.stop()
    logger.info("系统关闭")
```

### 5.2 启动顺序

```
1. Python main.py 执行
2. FastAPI 应用创建
3. lifespan 钩子触发
   ├─→ 初始化数据库
   ├─→ 启动采样服务
   ├─→ 启动自动交易引擎
   └─→ 启动 LOF011 同步
4. Uvicorn 监听 8000 端口
5. 接受 HTTP 请求
```

### 6.2 常见异常处理

| 异常 | 处理 |
|------|------|
| 通达信连接失败 | 降级到 QMT 或新浪 |
| QMT 端口占用 | 重启 Socket 世代（Generation） |
| 数据库锁 | WAL 模式支持并发读写 |
| API 超时 | 重试 3 次，间隔 2 秒 |
| 规则引擎崩溃 | 捕获异常，等待 10 秒继续 |
