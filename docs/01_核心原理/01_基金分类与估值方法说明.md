# 基金分类与估值方法详细说明

## 一、前后端分离的分类架构

### 1.1 设计原则

前端使用 **TAB 分类**（用户可见），后端使用 **估值方法分类**（程序计算）。一个前端 TAB 可以合并多个内部估值方法，但每种估值方法使用完全不同的算法。

### 1.2 前端 TAB 与内部估值方法映射

| 前端 TAB | 内部估值方法 | 跟踪标的 | 代表基金 | 数据源 |
|----------|-------------|---------|---------|--------|
| 黄金原油 | `commodity_gold_oil` | GLD、USO 及区域锚点 | 162411、162415 | 美股 ETF 价格 |
| QDII欧美 | `equity_us_etf` | 美股 ETF（SPY、QQQ 等） | 161126、161127、164906 | 美股 ETF 价格 |
| QDII欧美 | `equity_us_index` | 美国指数（.INX、.NDX） | 161125、161130 | **新浪指数价格** |
| QDII欧美 | `hybrid_cross` | 一篮子股票/ETF | 160225、501225 | 多资产组合价格 |
| QDII亚洲 | `equity_asia` | 亚洲市场指数/ETF | 161725、161726 | 亚洲市场数据 |
| 国内LOF | `lof_domestic` | A股 LOF 折溢价 | 501018、501025 等 | A股行情 |
| 白银 | `commodity_silver` | 上海期货交易所 | 161116 | 上期所白银期货 |

## 二、QDII欧美分类详解（重点）

### 2.1 合并逻辑

前端 TAB **"QDII欧美"** 合并了三个内部类别：
- **纯ETF** (`equity_us_etf`)：跟踪美股 ETF
- **美股指数** (`equity_us_index`)：跟踪美国指数
- **混合跨境** (`hybrid_cross`)：一篮子资产组合

**设计原因**：虽然估值算法不同，但都属于"投资美国市场"的 QDII 基金，用户只需关注一个 TAB。

### 2.2 纯ETF（equity_us_etf）

**代表基金**：161126、161127、164906

**跟踪标的**：美股 ETF
- 161126 → GLD（黄金 ETF）
- 161127 → USO（原油 ETF）
- 164906 → SPY/QQQ（标普 500/纳斯达克 ETF）

**估值算法**：`calculate_etf_valuation`
```python
# 使用 T-1 日净值 + 当日实时 ETF 价格
估值 = 净值 × (1 + 仓位 × ETF价格变化 × 汇率变动)
```

**数据源**：美股 ETF 实时价格（通过 Woody API、IB 等获取）

---

### 2.3 美股指数（equity_us_index）⭐ 特殊处理

**代表基金**：161125、161130

**跟踪标的**：美国指数（非 ETF）
- **161125（易方达标普 500）** → `.INX`（标普 500 指数）
- **161130（易方达纳斯达克 100）** → `.NDX`（纳斯达克 100 指数）

**特殊性**（⚠️ 必须保留的逻辑）：

1. **数据源不同**：
   - 指数价格从**新浪**获取（`.INX`、`.NDX`）
   - 而非跟踪 ETF 的价格
   - 代码实现：`static_valuation.py` 第 126-148 行

2. **时差处理**：
   - 中美交易时差约 **13-15 小时**
   - 使用 **T-1 日净值 + 当日实时指数价格**
   - 例如：2026-06-08（周一）A 股开盘时，美国 2026-06-07（周五）指数已收盘

3. **估值算法**：
   ```python
   # 指数标的估值（非 ETF）
   估值 = 净值 × (1 + 仓位 × (实时指数 / 基准指数 × 汇率变动 - 1))
   ```

4. **新浪指数 URL 解析**：
   ```python
   # 从基金配置中提取新浪指数 URL
   idx_url = fund.get('sina_index_url', '')
   # 解析格式：quotes/.INX.html → .INX
   m = re.search(r'(?:symbol=|list=gb_|quotes/)([.a-zA-Z0-9]+)', idx_url, re.IGNORECASE)
   idx_sym = f".{raw_sym}" if not raw_sym.startswith('.') else raw_sym
   ```

**代码实现**（`static_valuation.py` 第 126-148 行）：
```python
# 6. 匹配纯净指数行情 (如果有)
idx_url = fund.get('sina_index_url', '')
idx_sym = None
if idx_url:
    import re
    # 兼容新浪各种指数链接格式 (如 quotes/.INX.html)
    m = re.search(r'(?:symbol=|list=gb_|quotes/)([.a-zA-Z0-9]+)', idx_url, re.IGNORECASE)
    if m:
        raw_sym = m.group(1).upper().replace('.HTML', '')
        idx_sym = f".{raw_sym}" if not raw_sym.startswith('.') else raw_sym
        
if not idx_sym and category == '指数':
    trade_etf = str(fund.get('trade_etf', '')).upper()
    if 'QQQ' in trade_etf: idx_sym = '.NDX'
    elif 'SPY' in trade_etf: idx_sym = '.INX'
    
if idx_sym:
    idx_df = pd.read_sql(f'SELECT date, price as "{idx_sym}" FROM index_daily WHERE symbol = ?', 
                         conn, params=(idx_sym,))
    df = pd.merge(df, idx_df, on='date', how='left')
    # 强制使用纯净指数作为 primary_sym
    if category == '指数' and idx_sym in df.columns:
        primary_sym = idx_sym
```

### 2.4 混合跨境（hybrid_cross）

**代表基金**：160225、501225、164824

**跟踪标的**：一篮子股票/ETF

**估值算法**：`calculate_basket_valuation`
```python
# 矩阵公式估值
估值 = 净值 × [1 + 仓位 × Σ(权重_i × (现价_i / 基准价_i - 1)) × 汇率变动]
```

**示例**：
- **160225（西部利得海外研发）**：
  - GLD 40%（黄金 ETF）
  - XOP 30%（油气 ETF）
  - SLV 20%（白银 ETF）
  - 现金 10%

- **501225（工银印度）**：
  - 印度市场 ETF 组合

- **164824（工银印度）**：
  - 跟踪印度市场 ETF（区域锚点）

**数据源**：多资产组合价格（美股 ETF、亚洲 ETF 等）

## 三、黄金原油分类详解

### 3.1 跟踪标的

**代表基金**：162411（华宝油气）、162415

**跟踪标的**：
- **GLD**：黄金 ETF（跟踪黄金价格）
- **USO**：原油 ETF（跟踪原油价格）
- **区域锚点**：
  - XOP：油气 ETF（美国）
  - XBI：生物科技 ETF（美国）
  - 区域变种：GLD-JP（日本）、GLD-EU（欧洲）等

### 3.2 估值算法

```python
# 商品型估值（单一标的）
估值 = 净值 × (1 + 仓位 × (当前 ETF 价格 / 基准 ETF 价格 × 汇率变动 - 1))

# 多标的矩阵估值（一篮子商品）
估值 = 净值 × [1 + 仓位 × Σ(权重_i × (现价_i / 基准价_i - 1)) × 汇率变动]
```

### 3.3 数据源

- 美股 ETF 实时价格（GLD、USO、XOP 等）
- 通过 Woody API、IB（Interactive Brokers）等获取

## 六、⚠️ 时差问题：QDII欧美基金的共同挑战

### 6.1 核心问题：净值滞后 + 交易时差

**只有投资美国市场的QDII基金（QDII欧美）面临两个时间问题：**

1. **净值滞后（T+1或更晚）**：
   - 基金公司在**T日收盘后**计算净值
   - 但由于时差和数据收集，T日的净值通常在**T+1日或更晚**才能获取
   - 例如：2026-06-08（周一）A股开盘时，只能拿到2026-06-07（周五）的净值

2. **交易时差**：
   - 中美交易时差约 **13-15 小时**
   - A股开盘时，美股已经收盘约15小时
   - 例如：2026-06-08（周一）A股9:30开盘时，美国2026-06-07（周五）15:00已收盘

### 6.2 时差处理的通用公式

**QDII欧美基金的估值都遵循：**

```
估值 = T-1日净值 × (1 + 仓位 × 标的涨跌幅 × 汇率变动)
```

**关键**：这里的"T-1日净值"必须是**最新可获得的净值**，可能是1天、2天甚至更久之前。

### 6.3 各类基金的时差处理

#### 黄金原油（162411、162415）

- **跟踪标的**：GLD、USO、XOP（美股ETF）
- **时差处理**：
  - 净值：T-1日（可能滞后1-2天）
  - ETF价格：A股开盘时，美股已收盘约15小时
  - 例如：2026-06-08（周一）使用2026-06-07（周五）净值 + 2026-06-07（周五）ETF价格
- **数据源**：美股ETF实时价格（通过Woody API、IB等获取）

#### QDII欧美 - 纯ETF（161126、161127、164906）

- **跟踪标的**：SPY、QQQ、GLD等美股ETF
- **时差处理**：
  - 净值：T-1日（可能滞后1-2天）⭐ **需要识别净值日期**
  - ETF价格：A股开盘时，美股已收盘约15小时
  - 例如：2026-06-08（周一）使用2026-06-07（周五）净值 + 2026-06-07（周五）ETF价格
- **数据源**：美股ETF实时价格
- **⚠️ 注意**：虽然跟踪的是ETF，但净值同样滞后，必须仔细识别净值日期

#### QDII欧美 - 美股指数（161125、161130）

- **跟踪标的**：.INX（标普500）、.NDX（纳斯达克100）
- **时差处理**：
  - 净值：T-1日（可能滞后1-2天）⭐ **需要识别净值日期**
  - 指数价格：从新浪获取，A股开盘时美国已收盘约15小时
  - 例如：2026-06-08（周一）使用2026-06-07（周五）净值 + 2026-06-07（周五）指数价格
- **数据源**：新浪指数价格（非美股ETF）
- **代码实现**：`static_valuation.py` 第126-148行

#### QDII欧美 - 混合跨境（160225、501225、164824）

- **跟踪标的**：一篮子股票/ETF
- **时差处理**：
  - 净值：T-1日（可能滞后1-2天）⭐ **需要识别净值日期**
  - 标的价格：多市场数据，不同市场收盘时间不同
  - 例如：2026-06-08（周一）使用2026-06-07（周五）净值 + 2026-06-07（周五）各标的价格
- **数据源**：多资产组合价格（美股ETF、亚洲ETF等）
- **⚠️ 注意**：一篮子资产可能包含多个时区的标的，时差处理更复杂

#### QDII亚洲（161725、161726）

- **跟踪标的**：亚洲市场ETF/指数（如日本、香港等）
- **时差处理**：
  - ⚠️ **没有中美时差问题**
  - 净值：T-1日（可能滞后1-2天）⭐ **需要识别净值日期**
  - 亚洲市场：与A股市时差很小甚至没有时差
  - 例如：2026-06-08（周一）使用2026-06-07（周五）净值 + 2026-06-07（周五）亚洲市场数据
- **数据源**：亚洲市场数据
- **⚠️ 注意**：虽然没有时差问题，但净值仍然滞后，必须仔细识别净值日期

#### 国内LOF、白银

- **跟踪标的**：A股、上期所
- **时差处理**：
  - ✅ **没有时差问题**
  - 净值：T-1日（可能滞后1-2天）⭐ **需要识别净值日期**
- **数据源**：A股行情、上期所数据

### 6.4 净值日期识别的重要性

**⚠️ 核心问题**：由于净值滞后，系统读取到的净值日期可能与当前日期不一致。

**示例**：
```
当前日期：2026-06-08（周一）
最新可用净值日期：2026-06-07（周五）
净值滞后：1天（周末不交易）

如果2026-06-06（周五）基金暂停公布净值，则：
最新可用净值日期：2026-06-05（周四）
净值滞后：2天
```

**处理策略**：
1. **自动识别**：系统应自动读取最新可用的净值日期
2. **日志记录**：记录每次估值使用的净值日期和滞后天数
3. **异常告警**：当净值滞后超过2天时，发出告警
4. **估值标注**：在展示时标注估值使用的净值日期

### 6.5 代码实现建议

```python
# 获取最新可用净值
def get_latest_nav(fund_code, current_date):
    """
    获取最新可用的净值
    
    Args:
        fund_code: 基金代码
        current_date: 当前日期
    
    Returns:
        (nav_date, nav_value): 净值日期和净值
    """
    conn = get_db_connection()
    cursor = conn.cursor()
    
    # 查询最新可用净值（不超过当前日期）
    query = """
        SELECT nav_date, nav_value 
        FROM fund_nav 
        WHERE fund_code = ? AND nav_date <= ?
        ORDER BY nav_date DESC 
        LIMIT 1
    """
    cursor.execute(query, (fund_code, current_date))
    result = cursor.fetchone()
    
    if result:
        nav_date, nav_value = result
        lag_days = (current_date - nav_date).days
        print(f"基金 {fund_code} 使用 {nav_date} 的净值，滞后 {lag_days} 天")
        return nav_date, nav_value
    else:
        raise ValueError(f"基金 {fund_code} 找不到可用净值")
```
