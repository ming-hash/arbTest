# 程序3：ArbDashboard - 核心算法说明

## 一、核心业务逻辑定义

### 1.1 数据时间线

在 QDII 跨境基金套利中，存在三个关键时间节点：

| 术语 | 定义 | 数据来源 |
|------|------|----------|
| **基准日 (Benchmark Day)** | 基金公司公布净值的日期 | 东财 API |
| **估值日 (Valuation Day)** | 基于基准日净值 + 标的涨跌算出的估值日期 | 系统计算 |
| **今天 (Today)** | 实时盯盘日期 | 实时行情 |

### 1.2 核心公式

```
静态溢价率 = 当前现价 / 静态估值 - 1

实时溢价率 = 当前现价 / 实时估值 - 1
```

**关键区别**：
- **静态估值**：基于 `基准日净值 + 昨日收盘行情` 计算
- **实时估值**：基于 `基准日净值 + 实时行情` 计算

## 二、估值计算算法

### 2.1 魔法公式 (Magic Formula)

**文件**：`arbcore/calculators/valuation_math.py`

**用途**：单标的"魔法公式"估值，适用于单一底层资产的基金

**公式**：
```
估值 = 净值 × [1 + 仓位 × (标的价格变动 × 汇率变动 - 1)]
```

**代码实现**：
```python
def calculate_magic_valuation(base_nav, position, benchmark_change, fx_change):
    """
    魔法公式估值
    
    参数：
    - base_nav: 基准日净值
    - position: 底仓比例 (如 0.95)
    - benchmark_change: 标的价格变动 (如 0.02 表示涨 2%)
    - fx_change: 汇率变动 (如 -0.01 表示贬值 1%)
    
    返回：
    - 估值
    """
    valuation = base_nav * (1 + position * (benchmark_change - fx_change))
    return round(valuation, 4)
```

**示例**：
```python
# 162411 华宝油气（商品型 - 黄金原油）
base_nav = 1.150      # T-2 净值
position = 0.95       # 95% 底仓
benchmark_change = 0.02  # XOP 涨 2%
fx_change = -0.01     # 美元贬值 1%

valuation = calculate_magic_valuation(1.150, 0.95, 0.02, -0.01)
# = 1.150 × (1 + 0.95 × (0.02 - (-0.01)))
# = 1.150 × (1 + 0.95 × 0.03)
# = 1.150 × 1.0285
# = 1.1828
```

**注意**：不同基金类型使用不同估值方法：
- 商品型（162411）：`commodity_gold_oil` → 大宗商品期货价格
- 纯ETF型（161126）：`equity_us_etf` → 海外 ETF 价格
- **美股指数型（161125）**：`equity_us_index` → **新浪指数价格**（特殊处理）
- 混合跨境型（160225）：`hybrid_cross` → 一篮子资产价格

### 2.2 矩阵公式 (Portfolio Basket Formula)

**文件**：`arbcore/calculators/valuation_math.py`

**用途**：混合跨境（一篮子）矩阵估值，适用于多个底层标的的基金

**适用基金**：
- 160225 西部利得海外研发（混合跨境型）
- 161126 诺安全球黄金（纯ETF型，多资产组合）

**公式**：
```
估值 = 净值 × [1 + 仓位 × Σ(权重_i × (现价_i / 基准价_i - 1)) × 汇率变动]
```

**代码实现**：
```python
def calculate_basket_valuation(base_nav, position, basket_weights, asset_quotes):
    """
    矩阵公式估值
    
    参数：
    - base_nav: 基准日净值
    - position: 底仓比例
    - basket_weights: 权重映射 {asset_code: weight}
    - asset_quotes: 标的报价 {asset_code: {"current": price, "benchmark": price}}
    
    返回：
    - 估值
    """
    total_weighted_change = 0
    for asset_code, weight in basket_weights.items():
        if asset_code in asset_quotes:
            current = asset_quotes[asset_code]['current']
            benchmark = asset_quotes[asset_code]['benchmark']
            asset_change = current / benchmark - 1
            total_weighted_change += weight * asset_change
    
    valuation = base_nav * (1 + position * total_weighted_change)
    return round(valuation, 4)
```

**示例**：
```python
# 160225 西部利得海外研发（混合跨境型）
base_nav = 2.500
position = 0.90
basket_weights = {
    "GLD": 0.40,    # 黄金 ETF 40%
    "XOP": 0.30,    # 油气 ETF 30%
    "SLV": 0.20,    # 白银 ETF 20%
    "USD": 0.10     # 现金 10%
}
asset_quotes = {
    "GLD": {"current": 190.50, "benchmark": 188.00},  # 涨 1.33%
    "XOP": {"current": 85.20, "benchmark": 83.50},     # 涨 2.04%
    "SLV": {"current": 22.30, "benchmark": 22.00}      # 涨 1.36%
}

# 计算
weighted_change = 0.40 * (190.50/188.00 - 1) + 0.30 * (85.20/83.50 - 1) + 0.20 * (22.30/22.00 - 1)
             = 0.40 * 0.0133 + 0.30 * 0.0204 + 0.20 * 0.0136
             = 0.0053 + 0.0061 + 0.0027
             = 0.0141

valuation = 2.500 × (1 + 0.90 × 0.0141)
         = 2.500 × 1.0127
         = 2.5317
```

## 三、静态估值计算

**文件**：`arbcore/calculators/static_valuation.py`

**用途**：盘后计算每只基金的静态估值（基于收盘行情）

### 3.1 计算流程

```
1. 获取基金基本信息（基金代码、分类、持仓权重）
2. 获取基准日净值（T-2 或 T-1）
3. 获取当日收盘行情：
   - A 股基金收盘价
   - 海外 ETF 收盘价（XOP, GLD, USO 等）
   - 汇率（人民币中间价）
4. 调用 ValuationMath 计算估值
5. 计算溢价率 = (收盘价 - 估值) / 估值 × 100
6. 写入 unified_fund_history 表
```

### 3.2 核心代码

```python
class StaticValuationCalculator:
    def calculate(self, fund_code):
        """
        计算单只基金的静态估值
        
        优先级：
        1. 魔法捷径（单标的）
        2. 篮子权重模式（多标的）
        
        ⚠️ 时差处理：
        - 净值：T-1日或T-2日（可能滞后1-2天）
        - 行情：A股开盘时，海外市场已收盘约15小时
        - 必须识别净值日期，确保与行情日期匹配
        """
        # 1. 获取基金信息
        fund_info = self.get_fund_info(fund_code)
        
        # 2. 获取基准日净值（⚠️ 注意：可能是T-1或T-2日）
        nav_df = self.get_history_nav(fund_code)
        base_nav = nav_df.iloc[0]['nav']  # 最新净值（可能是1-2天前）
        nav_date = nav_df.iloc[0]['nav_date']  # 净值日期
        print(f"基金 {fund_code} 使用 {nav_date} 的净值: {base_nav}")
        
        # 3. 获取收盘行情
        a_share_price = self.get_realtime_quote(fund_code)['price']
        etf_quotes = self.get_etf_quotes(fund_info['basket_weights'])
        exchange_rate = self.get_exchange_rate()
        
        # 4. 计算估值
        if fund_info['type'] == 'SINGLE_ASSET':
            valuation = ValuationMath.calculate_magic_valuation(
                base_nav,
                fund_info['position'],
                etf_quotes['change'],
                exchange_rate['change']
            )
        else:
            valuation = ValuationMath.calculate_basket_valuation(
                base_nav,
                fund_info['position'],
                fund_info['basket_weights'],
                etf_quotes
            )
        
        # 5. 计算溢价率
        premium = (a_share_price - valuation) / valuation * 100
        
        return {
            'fund_code': fund_code,
            'date': today(),
            'price': a_share_price,
            'nav': base_nav,
            'static_val': valuation,
            'static_premium': premium
        }
```

## 四、实时估值计算

**文件**：`arbcore/calculators/dynamic_valuation.py`

**用途**：盘中实时计算基金的实时估值（基于实时行情）

### 4.1 计算流程

```
1. 获取基金基本信息
2. 获取基准日净值（缓存）
3. 获取实时行情（每秒更新）：
   - A 股基金现价
   - 海外 ETF 实时价
   - 实时汇率
4. 调用 ValuationMath 计算实时估值
5. 计算溢价率 = (现价 - 实时估值) / 实时估值 × 100
6. 写入 fund_intraday_quotes 表
```

### 4.2 核心代码

```python
class DynamicValuationCalculator:
    def __init__(self):
        self.nav_cache = {}  # 净值缓存
    
    def calculate(self, fund_code):
        """
        计算单只基金的实时估值
        """
        # 1. 获取缓存的净值
        if fund_code not in self.nav_cache:
            self.nav_cache[fund_code] = self.get_latest_nav(fund_code)
        
        base_nav = self.nav_cache[fund_code]['nav']
        position = self.nav_cache[fund_code]['position']
        
        # 2. 获取实时行情
        a_share_quote = self.get_realtime_a_share(fund_code)
        etf_quotes = self.get_realtime_etf_quotes()
        
        # 3. 计算实时估值
        valuation = ValuationMath.calculate_basket_valuation(
            base_nav,
            position,
            self.get_basket_weights(fund_code),
            etf_quotes
        )
        
        # 4. 计算溢价率
        premium = (a_share_quote['price'] - valuation) / valuation * 100
        
        return {
            'price': a_share_quote['price'],
            'rt_val': valuation,
            'premium': premium
        }
```

## 五、溢价率计算规范

### 5.1 静态溢价率

**公式**：
```
静态溢价率 = 当前现价 / 静态估值 - 1
```

**用途**：反映当前价格相对于"昨晚美股收盘锚点"的偏离度

**示例**：
```
现价 = 1.234
静态估值 = 1.180

静态溢价率 = 1.234 / 1.180 - 1 = 0.0458 = 4.58%
```

### 5.2 实时溢价率

**公式**：
```
实时溢价率 = 当前现价 / 实时估值 - 1
```

**用途**：反映当前价格相对于"实时估值"的偏离度

**示例**：
```
现价 = 1.234
实时估值 = 1.200

实时溢价率 = 1.234 / 1.200 - 1 = 0.0283 = 2.83%
```

### 5.3 涨跌幅计算

**公式**：
```
涨跌幅 = (现价 / 前一日收盘价 - 1) * 100
```

**数据来源**：
- **现价**：从 `fund_intraday_quotes`（实时采样表）获取，采样频率 60 秒/次
- **前一日收盘价**：从 `unified_fund_history`（历史表）获取

**示例**：
```
现价 = 4.302
前一日收盘价 = 4.250

涨跌幅 = (4.302 / 4.250 - 1) * 100 = 1.224%
```

---

### 5.4 关键区别

| 指标 | 比较基准 | 更新频率 | 用途 |
|------|----------|----------|------|
| 静态溢价率 | 静态估值（昨日收盘） | 随现价跳动 | 盘后总结 |
| 实时溢价率 | 实时估值（实时行情） | 随现价跳动 | 实时监控 |

**美股休市时**：实时估值 ≈ 静态估值，两个溢价率接近

**美股开盘后**：实时估值开始偏离静态估值，两个溢价率拉开差距

## 六、T+3 赎回提醒算法

**文件**：`backend/services/ledger_service.py`

**用途**：自动计算赎回提醒日（跳过周末）

### 6.1 算法

```python
from datetime import timedelta

def calculate_redeem_date(trade_date):
    """
    计算 T+3 工作日赎回提醒日
    
    规则：
    - 周一买 → 周四提醒
    - 周二买 → 周五提醒
    - 周三买 → 下周一提醒（跳过周末）
    """
    date = parse_date(trade_date)
    workdays_added = 0
    
    while workdays_added < 3:
        date += timedelta(days=1)
        if date.weekday() < 5:  # 周一到周五
            workdays_added += 1
    
    return date
```

### 6.2 测试用例

```python
# 周一买
calculate_redeem_date("2026-06-08")  # 周一 → 2026-06-11 (周四)

# 周二买
calculate_redeem_date("2026-06-09")  # 周二 → 2026-06-12 (周五)

# 周三买
calculate_redeem_date("2026-06-10")  # 周三 → 2026-06-15 (下周一)
```

### 8.2 美股指数的特殊处理（⚠️ 核心逻辑）

**代表基金**：161125（易方达标普 500）、161130（易方达纳斯达克 100）

**跟踪标的**：
- 161125 → `.INX`（标普 500 指数）
- 161130 → `.NDX`（纳斯达克 100 指数）

**特殊性**（必须保留的逻辑）：

1. **数据源不同**：
   - 指数价格从**新浪**获取（`.INX`、`.NDX`）
   - 而非跟踪 ETF 的价格
   - 代码实现：`static_valuation.py` 第 126-148 行

2. **时差处理**：
   - 中美交易时差约 **13-15 小时**
   - 使用 **T-1 日净值 + 当日实时指数价格**
   - 例如：2026-06-08（周一）A 股开盘时，美国 2026-06-07（周五）指数已收盘

3. **估值算法**：
   ```python
   # 指数标的估值（非 ETF）
   估值 = 净值 × (1 + 仓位 × (实时指数 / 基准指数 × 汇率变动 - 1))
   ```

4. **新浪指数 URL 解析**：
   ```python

   ```

### 8.1 网格化交易规则

**触发条件**：
```python
# 折价买入
if current_premium <= rule['threshold'] and current_position < rule['max_pos_wan']:
    execute_order('BUY', fund_code, rule['order_vol'], price)

    execute_order('SELL', fund_code, rule['order_vol'], price)
```

**示例规则**：
