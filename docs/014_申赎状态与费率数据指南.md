## 一、数据源

### 1.1 唯一数据源：东方财富（通过 AKShare）

```python
import akshare as ak
df = ak.fund_purchase_em()  # 全市场基金申赎数据，一次调用返回约 3 万条
```

**接口说明**：
- AKShare 封装的东财 `fund_purchase_em` 接口
- 每天只调用一次（防刷保护 `akshare_fund_status`）
- 返回全市场所有基金的申购/赎回状态、费率、限额

### 1.2 返回字段

| 东财原始字段 | 类型 | 说明 | 示例 |
|-------------|------|------|------|
| 基金代码 | TEXT | 6位数字代码 | 161124 |
| 申购状态 | TEXT | 申购状态文本 | 开放申购 / 限大额 / 暂停申购 |
| 赎回状态 | TEXT | 赎回状态文本 | 开放赎回 / 暂停赎回 |
| 日累计限定金额 | REAL | 每日累计申购上限（元） | 3000.0 / 500000.0 / 1e11 |
| 手续费 | TEXT | 申购费率 | 0.12 / 0.15 |
| 赎回费率 | TEXT | 赎回费率 | 0.50% |

> ⚠️ **关键发现**：东财的 `日累计限定金额` 字段包含了精确的限额数值！161124 的 3000 元 = 集思录显示的"限3千"。之前只取了文本"限大额"，没有取这个数值。

## 二、数据链路

```
东财 API (ak.fund_purchase_em)
    │  每日调用一次（防刷）
    ↓
arbcore/fetchers/data_fetcher.py → sync_akshare_fund_status()
    │  提取: fund_code, purchase_status, redemption_status,
    │        purchase_fee, redemption_fee, purchase_limit
    ↓  INSERT OR REPLACE
SQLite: fund_purchase_status 表
    │  fund_code (PK), purchase_status, redemption_status,
    │  purchase_fee, redemption_fee, purchase_limit, updated_at
    ↓  批量 SELECT
ArbDashboard/backend/services/fund_service.py
    │  get_unified_dashboard_data() 组装 JSON
    ↓  REST API
前端 Dashboard.vue
    │  formatPurchaseLimit() 格式化显示
    ↓
用户看到: 开放 | 限3千 | 限50万 | 暂停
```

## 三、数据库表结构

```sql
CREATE TABLE fund_purchase_status (
    fund_code        TEXT PRIMARY KEY,
    purchase_status  TEXT,          -- 申购状态（文本）
    redemption_status TEXT,         -- 赎回状态（文本）
    purchase_fee     TEXT,          -- 申购费率
    redemption_fee   TEXT,          -- 赎回费率
    purchase_limit   REAL,          -- 日累计申购限额（元）★ V1.0 新增
    updated_at       TIMESTAMP DEFAULT (datetime('now', 'localtime'))
);
```

## 四、申购状态枚举

### 4.1 状态文本与含义

| 东财原始文本 | 含义 | 前端标签色 | purchase_limit 典型值 |
|-------------|------|:--------:|:---------------------:|
| 开放申购 | 正常申购，无限制 | 🟢 success | 1e11（1000亿≈无限制） |
| 限大额 | 限制大额申购 | 🟡 warning | 3000 / 1000 / 500000 ... |
| 暂停申购 | 暂停申购 | ⚪ default | 极小值（10 / 5 / 1） |

### 4.2 限额格式化规则

**代码位置**：`Dashboard.vue` → `formatPurchaseLimit()`

| purchase_limit | 显示 | 规则 |
|:-------------:|------|------|
| ≥ 1亿 且 整除 1亿 | 限X亿 | `limit / 1e8` 亿 |
| ≥ 1万 且 整除 1万 | 限X万 | `limit / 1e4` 万 |
| ≥ 1000 | 限X千 | `Math.round(limit / 100) / 10` 千 |
| < 1000 | 限X | 直接显示数值 |
| null / 0 | 限大额 | 兜底（无限额数据时） |

**示例**：

| 基金 | purchase_limit | 前端显示 |
|------|:-------------:|---------|
| 161124 易方达香港小型股 | 3000 | 限3千 |
| 162415 华宝美国消费 | 1000 | 限1千 |
| 164824 工银印度 | 500000 | 限50万 |
| 161716 招商双债 | 5000000 | 限500万 |
| 164906 交银中证海外 | 1e11 | 开放（开放申购） |

### 4.3 前端渲染逻辑

**代码位置**：`Dashboard.vue` 申购列 render 函数

```typescript
// 伪代码
if (status.includes('开放')) {
    if (status.includes('限大额') || status.includes('限制')) {
        showText = formatPurchaseLimit(limit)   // "限3千"等
        tagType = 'warning'
    } else {
        showText = '开放'
        tagType = 'success'
    }
} else if (status.includes('暂停')) {
    showText = '暂停'
    tagType = 'default'
}
```

## 五、同步机制

### 5.1 自动同步（日更）

**位置**：`daily_updater.py` → `step4_5_sync_fund_purchase_status()`

每天 `daily_updater` 流程中自动执行一次，全量同步。

### 5.2 手动同步

```python
from arbcore.database.db_manager import DatabaseManager
from arbcore.fetchers.data_fetcher import data_fetcher

db = DatabaseManager(r'D:\Study\arbTest\database\arb_master.db')
# 如需强制刷新（清除防刷标记后）：
# conn.execute("DELETE FROM access_sync_status WHERE access_source = 'akshare_fund_status'")
data_fetcher.sync_akshare_fund_status(db)
```

### 5.3 防刷机制

- 检查表：`access_sync_status`
- 字段：`access_source = 'akshare_fund_status'`
- 每天最多调用一次东财 API
- 如需当天重跑，先删除该条记录

## 六、每日例行检查项

1. **申购状态是否更新**：QDII 基金申购状态可能随时变化，每日开盘前应确认
2. **限额是否合理**：对比集思录，确认"限大额"基金的限额数值正确
3. **暂停基金**：暂停申购的基金，日累计限定金额通常为极小值（1-100元），前端统一显示"暂停"
