## 二、估值算法体系

### 2.1 算法分类总览

系统采用**四种核心估值算法**，根据基金类型自动选择：

| 算法名称 | 适用范围 | 核心参数 | 代码位置 |
|---------|---------|---------|---------|
| **矩阵公式** | 有 basket 数据的基金（黄金原油、混合跨境） | position + basket 权重 + 汇率 | `valuation_math.py:calculate_basket_valuation` |
| **魔法公式** | 有 hedge 的单 ETF 基金（QDII欧美纯ETF/指数） | position + **hedge** + ETF 价格 + 汇率 | `valuation_math.py:calculate_magic_valuation` |
| **跟踪指数公式** | 无 Woody 数据的基金（QDII亚洲/国内LOF/现金管理） | position + 指数比率 + 汇率（无 hedge） | `daily_updater.py:step11_simple_static_valuation` |
| **日增量公式** | 期货跟踪基金（白银） | 期货价格变动 + 日度增量 | 特殊处理 |

### 2.2 矩阵公式（Basket Valuation）

**适用条件**：`fund_basket_weights` 表有记录的基金

**核心公式**：
```
实时估值 = NAV × (1 + pos × (Σ(price_ratio × weight) × fx_ratio - 1))
```

其中：
- `price_ratio = 当前实时价格 / 基准日收盘价`
- `weight = 资产权重（归一化）`
- `fx_ratio = 当前汇率 / 基准日汇率`
- `pos = 仓位（Woody API 每日更新）`

**Basket 构成示例**：

| 基金 | Basket 构成 | 权重 |
|------|------------|------|
| 501018（南方原油） | USO + ^USO-EU + ^USO-JP + ^USO-HK | 各约 25% |
| 501225（顺丰半导体） | SOXX(77%) + SZ159560(23%) | 77%/23% |
| 501312（海外科技） | ARKK + ARKG + ARKQ + ARKF | 多资产配置 |

**锚点（Anchor）概念**：
- 锚点是 Woody 的特殊符号，如 `^USO-EU`、`^GLD-JP`
- 实时估值分子：去掉 `^` 和区域后缀（-EU/-JP/-HK），取基础代码实时价格
- 分母：保留原始符号查数据库历史价格
- **只有 Woody API 认识锚点符号**，其他数据源不识别

### 2.3 魔法公式（Magic Valuation）

**适用条件**：
1. `fund_basket_weights` 无记录（单 ETF）
2. `fund_daily_factors.hedge` 有值（Woody API 返回）
3. `related_index` 指向美股 ETF（如 XOP、SPY）

**核心公式**：
```
实时估值 = NAV × (1 - pos) + (ETF实时价格 × 汇率) / hedge
```

**数学等价形式**：
```
实时估值 = NAV × (1 + pos × (etf_ratio × fx_ratio - 1))
```

其中：
- `hedge`：对冲比例（Woody API 每日返回，核心参数）
- `etf_ratio = ETF当前价格 / ETF基准日收盘价`
- `fx_ratio = 当前汇率 / 基准日汇率`

**Hedge 的作用**：
- Hedge 是基金经理设定的对冲比例，反映基金净值与标的价格的换算关系
- 例如 162411（华宝油气）的 hedge ≈ 1350，表示约 1350 份 XOP 对应 1 份基金净值
- **魔法公式必须有 hedge 才能计算**，无 hedge 时退化为仅显示 NAV

**典型案例**：
- 162411（华宝油气）→ XOP，hedge ≈ 1350
- 162415（美国消费）→ XLY，hedge ≈ 50
- 161125（标普500）→ SPY（代理.INX），hedge ≈ 500
- 161130（纳斯达克）→ QQQ（代理.NDX），hedge ≈ 400

### 2.4 跟踪指数公式（Simple Static Valuation）

**适用条件**：
- 无 Woody API 数据（QDII亚洲、国内LOF、现金管理）
- `related_index` 指向股票指数（如 HSI、399300、000012）

**核心公式**：
```
实时估值 = 前一交易日净值 × (1 + pos_ratio × (idx_ratio × fx_ratio - 1))
```

其中：
- `pos_ratio`：静态仓位比例（默认 95%，存储在 `unified_fund_list.pos_ratio`）
- `idx_ratio = 指数当前值 / 指数基准日收盘值`
- `fx_ratio = 当前汇率 / 基准日汇率`（A股无汇率=1）

**特点**：
- **无 hedge**：不使用 Woody API，无法获取精确对冲比例
- **静态仓位**：pos_ratio 默认为 95%，无每日更新
- **指数驱动**：完全依赖指数价格变动推算净值

**不同市场的汇率处理**：

| 基金类型 | 汇率 | 说明 |
|---------|------|------|
| QDII亚洲 | 港币汇率 | 跟踪港股指数需港币汇率 |
| 国内LOF | 无（=1） | A股基金无需汇率 |
| 现金管理 | 无（=1） | 债券指数无汇率 |

### 2.5 日增量公式（白银期货）

**适用条件**：仅 161226（国瑞白银期货）

**核心逻辑**：
- 跟踪上海期货交易所白银期货主力合约
- 使用日度增量法：基期净值 + 期货价格变动 × 持仓比例

### 2.6 ⚠️ 时差问题：QDII 基金的共同挑战

**只有投资美国市场的 QDII 基金（QDII欧美）面临两个时间问题：**

#### 2.6.1 核心问题：净值滞后 + 交易时差

1. **净值滞后（T+1 或更晚）**：
   - 基金公司在**T 日收盘后**计算净值
   - 但由于时差和数据收集，T 日的净值通常在**T+1 日或更晚**才能获取
   - 例如：2026-06-08（周一）A 股开盘时，只能拿到 2026-06-07（周五）的净值

2. **交易时差**：
   - 中美交易时差约 **13-15 小时**
   - A 股开盘时，美股已经收盘约 15 小时
   - 例如：2026-06-08（周一）A 股 9:30 开盘时，美国 2026-06-07（周五）15:00 已收盘

#### 2.6.2 时差处理的通用公式

**QDII 欧美基金的估值都遵循：**

```
估值 = T-1 日净值 × (1 + 仓位 × 标的涨跌幅 × 汇率变动)
```

**关键**：这里的"T-1 日净值"必须是**最新可获得的净值**，可能是 1 天、2 天甚至更久之前。

#### 2.6.3 各类基金的时差处理

| 基金类型 | 净值日期 | 标的价格 | 说明 |
|---------|---------|---------|------|
| 黄金原油 | T-1 日（可能滞后 1-2 天） | 美股 ETF 收盘 | 净值 + ETF 价格都来自上周五 |
| QDII欧美 - 纯ETF | T-1 日（可能滞后 1-2 天） | 美股 ETF 收盘 | 同上 |
| QDII欧美 - 美股指数 | T-1 日（可能滞后 1-2 天） | 新浪指数收盘 | 同上 |
| 混合跨境 | T-1 日（可能滞后 1-2 天） | 多市场收盘 | 一篮子资产可能包含多个时区 |
| QDII亚洲 | T-1 日（可能滞后 1-2 天） | 港股收盘 | 无中美时差，但净值仍滞后 |
| 国内LOF | T-1 日（可能滞后 1-2 天） | A 股收盘 | 无时差问题 |
| 白银 | T-1 日（可能滞后 1-2 天） | 期货收盘 | 无时差问题 |

#### 2.6.4 净值日期识别的重要性

**⚠️ 核心问题**：由于净值滞后，系统读取到的净值日期可能与当前日期不一致。

**处理策略**：
1. **自动识别**：系统应自动读取最新可用的净值日期
2. **日志记录**：记录每次估值使用的净值日期和滞后天数
3. **异常告警**：当净值滞后超过 2 天时，发出告警
4. **估值标注**：在展示时标注估值使用的净值日期

## 三、估值算法分流规则

### 3.1 整体分流流程

```
基金配置
    │
    ├─ 有 Woody API 数据（黄金原油/QDII欧美）
    │       │
    │       ├─ fund_basket_weights 有记录 → 矩阵公式
    │       │       │
    │       │       └─ 黄金原油（多锚点）/ 混合跨境（多资产）
    │       │
    │       └─ fund_basket_weights 无记录 → 魔法公式
    │               │
    │               └─ QDII欧美纯ETF / QDII欧美指数
    │
    └─ 无 Woody API 数据（QDII亚洲/国内LOF/现金管理）
            │
            └─ 跟踪指数公式
                    │
                    └─ QDII亚洲（港股指数）/ 国内LOF（A股指数）/ 现金管理（债券指数）
```

### 3.2 分流判断代码逻辑

**代码位置**：`LOF005_output.py` 约 L200-217

```python
# 识别是否启动魔术算法
is_magic_valuation = False

# 单一ETF基金（没有basket_weights但有related_index和hedge）使用魔术算法
# 多ETF组合使用矩阵算法
if (len(etfs) == 1 or is_single_etf) and current_hedge != "":
    try:
        if float(current_hedge) > 0:
            is_magic_valuation = True
    except:
        pass

# 魔术算法计算实时估值
if is_magic_valuation:
    row_data.append(f'=IFERROR(G{base_r}*(1-B{r}) + K{r}*C{r}/{hedge_col_letter}{r}, "")')
else:
    # 矩阵算法计算实时估值
    row_data.append(f'=IFERROR(G{base_r}*(1+B{r}*(({etf_sum})*(C{r}/C{base_r})-1)), "")')
```

**分流判断条件优先级**：
1. **Basket 数据优先**：有 basket → 矩阵公式（无论是否有 hedge）
2. **Hedge 兜底**：无 basket 但有 hedge → 魔法公式
3. **指数兜底**：无 basket 无 hedge → 跟踪指数公式

## 四、指数数据源梳理

### 4.1 整体策略概览

#### 历史数据（`index_history` 表）

**数据源：tqcenter（通达信客户端本地 DLL）**

- 前提：通达信客户端必须已打开
- 脚本：`scripts/backfill_tdx_index.py`
- 符号格式：`399300.SZ`（深圳）、`000905.SH`（上海）、`HSI.HI`（港股）
- 覆盖范围：2025-01-01 起，约 353 个交易日
- **不覆盖**：美股 ETF（XOP、GLD、QQQ 等）— TDX 没有这些数据，美股历史走 Woody API / IB

#### 实时数据（`index_realtime_quotes` 表）

**A 股指数**（399xxx、000xxx）— 瀑布流四级降级：
```
① tqcenter.get_market_snapshot()  ← 主力（通达信本地DLL，毫秒级）
② 腾讯财经 API (qt.gtimg.cn)      ← 备用
③ 新浪 API                        ← 兜底
④ 东财 API                        ← 最后兜底
```

**港股指数**（HSI、HSCEI、HSTECH 等）— 瀑布流四级降级：
```
① 腾讯 API (hkHSI 格式)           ← 主力（仅支持 HSI/HSCEI/HSTECH）
② 新浪 API (rt_hkHSI 格式)        ← 备用（同上）
③ 东财 API (secid格式)            ← 兜底（覆盖 HSCI/HSMI/HSSI/HSCCI/HSSCNE/CSI系列）
④ tqcenter.get_market_snapshot()  ← 最后兜底（港股数据不可靠）
```

**美股 ETF**（XOP、GLD、QQQ 等）— 不走指数通道：
```
① IB (ib_reader.py)               ← 主力
② 富途 (futu_reader.py)           ← 备用
```

#### 数据流向

```
历史数据（盘后回溯）：
  backfill_tdx_index.py → tqcenter DLL → index_history 表 → 用于估值回溯、溢价计算

实时数据（盘中高频）：
  tqcenter/腾讯/新浪/东财 → index_realtime_quotes 表 → 实时估值面板
```

核心逻辑在 `jsl/01_rt_all.py` 的 `_fetch_index_realtime()` 方法中，A 股和港股走不同分支，美股 ETF 完全不经过指数通道。

程序3（ArbDashboard）的实时指数预取在 `ArbDashboard/backend/services/fund_service.py` → `prefetch_index_changes()`，采用三级瀑布流：腾讯 → 新浪 → 东财。

## 七、东财 API 获取港股指数 — 技术细节

### 7.1 为什么需要东财兜底？

腾讯/新浪港股接口仅支持 3 个恒生核心指数（HSI、HSCEI、HSTECH），以下指数**无法获取**：

| 指数 | 类型 | 问题 |
|------|------|------|
| HSCI, HSMI, HSSI | 恒生综合/中型股/小型股 | 腾讯/新浪不识别 |
| HSCCI, HSSCNE | 恒生综合公用/消费 | 腾讯/新浪不识别 |
| 930914.CSI, 930917.CSI | 中证港股指数 | 腾讯/新浪不识别（.CSI后缀） |

### 7.2 东财 API 请求格式

**接口地址**：`https://push2.eastmoney.com/api/qt/stock/get`

**请求参数**：

| 参数 | 值 | 说明 |
|------|-----|------|
| `secid` | `{market}.{code}` | 东财内部编码，见下方映射表 |
| `fields` | `f43,f58,f170` | f43=最新价, f58=名称, f170=涨跌幅 |
| `ut` | `fa5fd1943c7b386f172d6893dbfba10b` | 固定 token |
| `fltt` | `1` | 返回原始值（不除以100） |

**响应格式**（JSON）：
```json
{
  "rc": 0,
  "data": {
    "f43": 23275,      // 最新价（已×100，需/100？不，fltt=1时是原始值）
    "f58": "恒生指数",  // 名称
    "f170": -208        // 涨跌幅（已×100，需/100？不，fltt=1时是原始值）
  }
}
```

**请求头**：
```python
headers = {
    'Referer': 'https://quote.eastmoney.com/',
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36',
}
```

### 7.3 secid 映射规则

东财对不同市场/系列使用不同的 `market` 编号：

| 指数类型 | secid 格式 | market 编号 | 示例 |
|---------|-----------|:-----------:|------|
| 恒生指数 HSI | `116.HSI` | 116 | `116.HSI` |
| 恒生国企 HSCEI | `100.HSCEI` | 100 | `100.HSCEI` |
| 恒生综合 HSCI | `116.HSCI` | 116 | `116.HSCI` |
| 恒生中型股 HSMI | `124.HSMI` | 124 | `124.HSMI` |
| 恒生小型股 HSSI | `124.HSSI` | 124 | `124.HSSI` |
| 恒生综合公用 HSCCI | `116.HSCCI` | 116 | `116.HSCCI` |
| 恒生综合消费 HSSCNE | `124.HSSCNE` | 124 | `124.HSSCNE` |
| 中证港股通高股息 930914 | `2.930914` | 2 | `2.930914` |
| 中证SHS高股息 930917 | `2.930917` | 2 | `2.930917` |

**映射规则总结**：
```python
EM_SECID_MAP = {
    'HSSI': '124.HSSI', 'HSMI': '124.HSMI',
    'HSSFML25': '124.HSSFML25', 'HSSCBBAI': '124.HSSCBBAI',
    'HSCEI': '100.HSCEI',
}
EM_HK_KEYWORDS = {'HSI', 'HSCI', 'HSCCI', 'HSSCNE', 'HSTECH'}

def get_eastmoney_secid(symbol):
    if symbol in EM_SECID_MAP:
        return EM_SECID_MAP[symbol]
    elif symbol[:3] == 'CSI':
        return f"2.{symbol[3:]}"       # CSI前缀 → 2.代码
    elif symbol.startswith('H') and symbol in EM_HK_KEYWORDS:
        return f"116.{symbol}"          # 默认港股 → 116.代码
    elif symbol.endswith('.CSI'):
        code = symbol[:-4]
        if code.isdigit():
            return f"2.{code}"          # 930914.CSI → 2.930914
    return None
```

## 九、关键发现

### 9.1 TDX（通达信）是最强的历史数据源

- 支持所有A股指数
- 支持港股指数
- **支持中证非标CSI代码**（930713、H30094等）！
- 需要打开通达信客户端，通过本地 DLL (`TPythClient.dll`) 读取

### 9.2 新浪港股格式与A股不同

A股格式：`s_sz399300="深证成指,价格,涨跌额,涨跌幅%,成交量,成交额"`
港股格式：`rt_hkHSI="HSI,恒生指数,开盘,昨收,最高,最低,当前价,涨跌额,涨跌幅%,..."`

解析港股时：
- 价格：`parts[6]`
- 涨跌幅：`parts[8]`

### 9.3 美股ETF不应出现在related_index里

美股ETF（XOP、GLD、USO等）应该走IB/Futu专门接口，不应该被当作A股指数通过腾讯/新浪获取。

### 9.4 东财API是港股非标指数的唯一实时数据源

恒生综合系列（HSCI/HSMI/HSSI/HSCCI/HSSCNE）和中证港股指数（930914/930917）只能通过东财API获取实时价格。
