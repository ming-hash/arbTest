# lof_config.yaml — 基金核心配置说明

> **适合读者**：新手用户、非技术背景用户。本文档解释什么是 `lof_config.yaml`、它有什么用、以及如何安全地修改它。

---

## 一、这是什么文件？

`lof_config.yaml` 是套利监控系统的**核心配置文件**，记录了所有监控基金的"身份证"信息。

**一句话概括**：它告诉系统"监控哪些基金、每只基金跟踪什么标的、仓位多少、属于哪个分类"。

没有这个文件，系统就不知道要监控哪些基金，套利看板和实时估值都无法工作。

arbcore/config/lof_config.yaml

## 三、文件结构概览

整个文件分为两大部分：

```yaml
currencies:        # 货币汇率配置（系统级）
  - currency: GBP
    symbol: USD

funds:             # 基金列表（核心内容）
  - code: '501018'
    name: 南方原油
    category: 黄金原油
    ...（更多字段）
  - code: '161125'
    name: 易方达标普500
    category: QDII欧美
    ...
```

## 四、每只基金的字段说明

以「南方原油（501018）」为例，逐字段解释：

| 字段 | 示例值 | 说明 |
|------|--------|------|
| `code` | `"501018"` | 基金代码，唯一标识，不能重复 |
| `name` | `"南方原油"` | 基金名称，显示用 |
| `category` | `"黄金原油"` | 所属分类，决定在前端哪个 TAB 下显示 |
| `sub_category` | `"原油"` | 子分类，决定估值算法 |
| `position` | `88.28` | 当前仓位（百分比），来自 Woody API 自动更新 |
| `trade_etf` | `"USO"` | 交易跟踪的 ETF 代码 |
| `trade_future` | `"MCL"` | 对冲期货代码（可选） |
| `related_index` | `"USO"` | 关联指数/标的 |
| `woody_supported` | `true` | 是否支持 Woody API 估值 |
| `data_source` | `"woody"` | 数据源（woody / sina / tdx 等） |

### 4.1 持仓相关字段

| 字段 | 说明 |
|------|------|
| `holdings` | 持仓比例，包含 `equity_ratio`（股票仓位）和 `cash_ratio`（现金比例） |
| `holdings_portfolio` | 基金实际持仓明细（各底层 ETF 的权重） |
| `valuation_portfolio` | 估值篮子：用于计算实时估值的 ETF 权重分配 |
| `future_hedging` | 期货对冲信息（如有），包含期货代码、交割月份、权重等 |

### 4.2 holdings_portfolio 子字段

```yaml
holdings_portfolio:
  - symbol: USO         # ETF 代码
    weight: 17.22       # 权重（%）
    currency: USD       # 交易货币
    exchange: LSE       # 交易所
    name: United...     # ETF 全名
```

### 4.3 valuation_portfolio 子字段

```yaml
valuation_portfolio:
  - symbol: USO         # ETF 代码
    weight: 50.79       # 估值权重（%）
    anchor: US          # 锚点区域（US/EU/JP/HK）
```

> ⚠️ 注意：`holdings_portfolio` 是**真实持仓**，`valuation_portfolio` 是**估值用的一篮子 ETF**，二者可以不同。Woody API 会根据真实持仓自动生成估值篮子。

## 五、如何在页面上管理？

### 5.1 找到入口

1. 打开套利看板系统（ArbDashboard）
2. 左侧菜单 → **数据管理**
3. 找到「**核心基金配置**」卡片

### 5.2 页面功能

| 功能 | 操作 | 说明 |
|------|------|------|
| **查看** | 页面自动加载 | 顶部下拉框可按分类筛选（黄金原油/QDII欧美等） |
| **编辑** | 点击基金卡片 | 弹出编辑窗口，修改各项参数后保存 |
| **删除** | 点击基金卡片中的删除按钮 | 从配置中移除该基金 |
| **快速添加** | 输入基金代码 → 回车 | 快速创建新基金，补充信息后保存 |

## 六、导入 / 导出功能（新增）

这是本次改造新增的功能，位于「核心基金配置」卡片右上角的两个按钮。

### 6.1 导出（备份配置）

**点击「导出」按钮** → 浏览器自动下载一个带时间戳的 YAML 文件。

```
文件名示例：lof_config_20260625_223000.yaml
```

用途：
- ✅ 修改配置前先导出备份，改坏了可以恢复
- ✅ 把配置分享给其他人
- ✅ 在不同环境之间迁移配置

### 6.2 导入（恢复/替换配置）

**点击「导入」按钮** → 选择 `.yaml` 文件 → 确认导入。

流程：
1. 弹出导入窗口，提醒"导入将覆盖当前所有基金配置"
2. 点击「选择 YAML 文件」，选一个之前导出的备份文件
3. 确认文件后，点击「确认导入」

安全机制：
- 📦 导入前**自动备份**旧配置为 `lof_config.yaml.bak`（同目录下）
- ✅ 导入前**验证 YAML 结构**（必须有 `funds` 字段才算合法）
- ✅ 如果验证失败，导入会被拒绝，旧配置不受影响

### 6.3 典型使用场景

| 场景 | 操作 |
|------|------|
| 批量修改基金配置前 | 先**导出**备份 → 放心改 → 改坏了就**导入**恢复 |
| 新增了一批基金 | 导出保存 → 以后可以恢复到当前状态 |
| 从别人那拿到配置 | 导入对方的 YAML → 立刻拥有全部基金配置 |
| 配置损坏/乱码 | 导入最近的正常备份 → 秒恢复 |

## 七、什么是 YAML 格式？（新手必读）

YAML 是一种纯文本的配置文件格式，用**缩进**表示层级关系，就像写大纲一样：

```yaml
# 这是注释（以 # 开头）
funds:
  - code: '501018'      # 缩进 2 个空格表示属于 funds
    name: 南方原油       # 缩进 4 个空格表示属于当前基金
    category: 黄金原油   # 同样缩进 4 个空格
```

**铁律**：
- ⚠️ 缩进必须用**空格**，不能用 Tab
- ⚠️ 同级字段缩进必须一致（推荐 2 个空格）
- ⚠️ 字符串可以不加引号，但代码类最好加：`'501018'`

> 💡 建议通过页面 UI 修改配置，不要手动编辑 YAML 文件。导入导出功能就是为了避免手误。

## 八、安全红线

| 规则 | 说明 |
|------|------|
| 🔒 `lof_config.yaml` **不会上传 GitHub** | `.gitignore` 已排除，涉密信息绝对安全 |

| ⚠️ 不要手动编辑 | 缩进错一个空格整个文件就坏了，用页面 UI 改 |
| 📋 定期导出 | 建议每次改完配置后导出一份，保存到私人文件夹 |

## 九、常见问题

**Q：为什么不用 Excel？**
A：YAML 是程序直接读取的标准格式，速度快、零依赖、不挑操作系统。Excel 需要安装解析库，增加复杂度。

**Q：能直接新增基金吗？**
A：可以。在「核心基金配置」页面输入基金代码回车即可。但建议先在数据库 `unified_fund_list` 表中确认该基金有对应的 basket/因子数据。

**Q：导入后前端显示不变？**
A：导入只是更新文件，前端需要刷新页面才会重新加载。另外确认分类字段 `category` 是否写对了。

**Q：导入报错怎么办？**
A：最常见原因是 YAML 格式不对（缩进错误）。检查报错信息，或者用在线 YAML 校验工具检查文件。
